File Access and I/O Investigation Report
1. Open Files & File Descriptors

Command: ls -l /proc/$$/fd

Observation: Accessed the /proc filesystem to view open file descriptors for the current shell. The output shows file descriptors 0 (stdin), 1 (stdout), 2 (stderr), and 255, which confirms that the kernel is actively managing terminal I/O for this process.

2. Redirection Results

Commands: echo "Log Message" > log.txt and ls non_existence_file 2> error.txt

Observation: Standard output was successfully redirected to log.txt. The error generated by the ls command was captured by standard error descriptor 2 and saved into error.txt, demonstrating how Linux separates data and error streams.

3. Resource Limits

Command: ulimit -a

Observation: The output listed system-wide constraints on process resources. Notably, open files (-n) 1048576 indicates the maximum number of file descriptors a single process can have open, which is essential for load management.

4. How Linux Manages File I/O

Linux follows the "Everything is a file" philosophy. When an application requests I/O, the kernel:

Opens the file and assigns it a File Descriptor (a non-negative integer).

Maps this descriptor to an Inode on the disk.

Uses system buffers to optimize data flow between user space and physical storage.
